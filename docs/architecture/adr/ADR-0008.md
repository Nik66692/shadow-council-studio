# ADR-0008 — canonical import evidence, review drafts and human-approved canon

Status: Accepted

## Context

Phase 1 must make the approved Source of Truth inspectable without turning document parsing into an autonomous design authority. DOCX structure is imperfect evidence: headings, paragraphs, lists and tables can be extracted, but typography alone cannot safely determine canonical meaning or status.

## Decision

Use three strictly separated layers.

### Raw extraction evidence

`canon_raw_blocks` stores text extracted from the manifest-selected DOCX together with deterministic source anchors, source part, document order, block kind, style metadata and SHA-256 of the extracted text.

Raw blocks are evidence only. They preserve the extracted Italian text and never receive a canonical status.

### Normalized review drafts

`canon_normalized_drafts` provides a one-to-one, read-only review representation of raw blocks. Every imported draft starts as `PENDING_HUMAN_REVIEW` and `canonical_status` remains `NULL`.

Normalization in Phase 1 may classify only structural block kinds such as heading, paragraph, list item or flattened table text. It must not summarize, translate, rewrite or interpret game rules.

### Human-approved canon

Approved canonical entities are explicitly outside Phase 1. A later workflow may create source-linked canonical records only after Niccolò reviews and approves them.

The import pipeline must never promote drafts automatically, infer canon status from typography or modify the canonical DOCX.

## Determinism and provenance

The current source is selected only through `docs/canon/source/manifest.json` and its SHA-256 is verified before parsing.

Import runs are idempotent for the pair `(source_sha256, importer_version)`. Source anchors include source version, source hash, DOCX part, structural block kind and document-order index.

Unsupported or lossy structures generate persisted, visible warnings instead of silent omission.

## Application boundary

SQLite remains behind Rust repository and service functions. React accesses import evidence only through typed Tauri commands and receives a read-only review snapshot.

## Consequences

- The database can prove where every imported draft came from.
- Repeated imports do not duplicate evidence.
- Tables and unsupported structures remain visible as review risks.
- Phase 1 cannot accidentally redefine Shadow Council canon.
- A later approval model can be added without rewriting raw import history.

## Alternatives considered

- Import headings directly as canonical rules.
- Infer canonical status from document formatting.
- Store only rendered HTML without structured provenance.
- Parse and silently discard unsupported DOCX structures.

All were rejected because they weaken traceability or allow automation to exceed its authority.

## Review triggers

- A human-approval workflow is designed.
- The Source of Truth format changes materially.
- Additional DOCX parts or richer table structures must be preserved.
